# MLX Workshop Beta Support Matrix

This matrix separates a working product path from fixtures, historical evidence,
and deliberately unsupported claims. `adapter-required` is a supported fail-closed
inspection result.

| Path | Beta evidence | Classification | Resource rule |
| --- | --- | --- | --- |
| Generated tiny dense Llama safetensors model | Inspect, plan, real MXFP4 conversion, deterministic load/generation, parent immutability, three qualification gates | Required real reference campaign; no quality claim | Tiny and reproducible; safe for CI/development |
| Deterministic workflow fixtures | Streaming, failure, cancellation, interruption, relaunch recovery, resume, recovered cancellation, and qualification routing | Required lifecycle integration evidence | No meaningful model load |
| Real host snapshot | Chip, memory, free disk, tool versions, power/thermal context, and sanitized MLX-related workload names | Wired and measured | Read-only; never stops workloads |
| Dense Llama uniform MXFP4/MXFP8/affine conversion | Guided setup and plan review, host-fit estimates, exact MLX-LM commands, explicit confirmation, immutable output, and verification | Supported when capability inspection and resource review pass; real tiny-model route is covered by native XCUITest | Every real run requires current confirmation |
| Calibration-driven mixed precision | Canonical module IDs, calibration contract, protection rules, search and Pareto reduction | Deterministic library evidence only | Never display fixture deltas as model measurements |
| Refusal-direction behavior editing | Dataset separation, provenance, adapter checks, and promotion contract | Demo/contract only; not executable in live UI | No behavior-edit smoke until adapters and sealed data gates pass |
| Qualification | Frozen gates, raw-evidence hashes, parent/candidate snapshots, and parent parity | Wired for completed supported runs | Pending or failed gates remain unqualified |
| Compare, Behavior, and Sensitivity UI | Design and domain fixtures | Demo-only; hidden from live navigation | Never present synthetic values as live evidence |
| Vision, MTP, native FP8, unknown tensor semantics, or unknown architectures | Explicit blocker with adapter guidance | Supported fail-closed outcome | No conversion until an adapter and reference fixture pass |

Beta v1 does not claim universal model compatibility, generic behavior editing,
vision/MTP packaging, canonical-model replacement, or model quality. Distribution
also remains gated on Developer ID signing and notarization as recorded in
[../BETA_REPORT.md](../BETA_REPORT.md).
